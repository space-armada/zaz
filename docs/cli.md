## Global flags

The flags below apply to every subcommand: `-c/--config`, `-s/--socket`,
`--debug`, `--log-file`. The `--full`, `--multi-pane`, `--no-autostart`, and
`--stop-on-exit` flags are TUI-mode-only and are documented under
[zaz](#zaz-default-tui-mode).

To be written in milestone 23.5.

## Socket and config resolution

Every subcommand resolves the daemon socket and project config the same way:
walk up from CWD looking for `zaz.toml` or `zaz.json`, derive the socket from
that config, allow `--socket` and `--config` to override. `--socket` wins over
`--config` when both are passed. The MCP page ([mcp.md](mcp.md)) defers to
this section.

To be written in milestone 23.5.

## Exit codes

zaz follows the LSB / `systemctl` convention:

- `0` — success or idempotent no-op
- `1` — operational error
- `3` — "not running" (queried daemon is absent)

See ADR-0004 and ZAZ-005 for the rationale.

To be written in milestone 23.5.

## Log files and rotation

zaz writes daemon log files under `$XDG_STATE_HOME/zaz/` when set, otherwise
`~/.local/state/zaz/`. The default file set:

- `daemon-output.log` — always written for daemons launched via TUI autostart
  or `zaz start`; captures panics and pre-init errors.
- `tui-debug.log` and `daemon-debug.log` — written when `--debug` is passed in
  TUI mode.
- A user-provided `--log-file <path>` overrides the TUI debug file; the
  autostarted daemon uses a sibling `*.daemon.log` file derived from that
  path.

Rotation: 10 MB per file, 5 generations, 200 MB total budget; oldest rotated
files are pruned first when the budget is exceeded.
